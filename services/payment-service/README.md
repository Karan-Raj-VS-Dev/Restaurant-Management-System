# payment-service

## Purpose

`payment-service` manages payment processing and payment result tracking for dine-in and takeaway orders.

## Default Port

- `8091`

## Current APIs

- `POST /api/payments`

## Current Scope

- payment processing placeholder
- support for cash, card, UPI, and wallet methods

## Planned Responsibilities

- payment persistence
- gateway integration
- payment retries and failure handling
- refund support
- cash reconciliation support
- publication of payment success and failure events
